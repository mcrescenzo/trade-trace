### The bitcoin-$1m market: memory + resolution discipline working together

I bound `Will bitcoin hit $1m before GTA VI?` (gamma 540844, `mkt_5hSASzEygEe5WSvv`)
intending a "strong NO" forecast. Two things stopped that mistake:

1. `market.bind`'s `venue_metadata_json.description` carried the **full
   resolution rule**, which contains a **50-50 fallback**: if *neither*
   bitcoin-$1m *nor* a GTA VI release occurs by 2026-07-31, the market resolves
   50-50 — not NO. So the ~0.4925 token mid is dominated by GTA-VI release
   timing + the 50-50 split, and is **not** comparable to P(literal YES) ≈ 0.02.
2. `memory.recall` returned `mem_6wWb9uY0nF3qaJ29` (importance 7) warning that
   "X before GTA VI" mids cluster near 0.50 and don't track P(X) — treat as
   resolution-ambiguous, skip or probe.

Honest play: forecast the *literal* YES event at 0.02 (`very_low` confidence,
rationale flagging the non-comparable mid), record a resolution interpretation
documenting the 50-50 clause, then **skip** on `forecast_ambiguous` grounds, and
reflect the generalized lesson (`mem_Ie3aqoD7t8v5a9JG`).

## Fix detail (AX-019)

- Gamma `/markets` has no free-text search; its real search endpoint is
  `/public-search`, which returns an **event-centric** shape
  (`events[].markets[]`).
- The query path now routes to `/public-search`, flattens nested markets through
  the existing `_market_search_candidate` projection, and drops `closed` markets
  client-side (the `/markets` no-query path already filtered them upstream). The
  no-query path is unchanged.
- New helpers `_extract_market_rows` / `_gamma_public_search_rows`.
- Tests: rewrote the query-path integration test to assert the corrected
  contract (query → `/public-search`, never `/markets`; events flattened) and
  added a closed-filter test. **No test was weakened** — the old test asserted
  the buggy `/markets?q=` behavior and now asserts the fix with stronger checks.
- **Live-verified** via `tt` CLI (editable install, fresh process):
  `--query bitcoin` → bitcoin-price markets; `--query "taylor swift"` → Swift
  markets. Live MCP picks this up next fresh server boot.
