### NEW — `report.calibration_advisory` low-N `suggested_probability` foot-gun → filed `trade-trace-suit`

- Surface: `report.calibration_advisory` output contract. Severity: low–medium
  (the tool warns; the trap is for naive programmatic consumers).
- Live evidence: candidate YES 0.88 for "BTC above $60k" (market 0.8925, BTC
  above the strike). Band 0.8–0.9 had **N=1** (one prior near-money YES that
  settled NO, Brier 0.7225). Tool returned `suggested_probability=0.03`,
  `suggested_adjustment=-0.85`, with `sample_warning='only 1 prior forecast(s) …
  unreliable below 20'`. Acting on `suggested_probability` blindly would
  re-forecast an above-the-money market down to 0.03.
- The asymmetry (crux): at `sample_size==0` the code nulls `suggested_probability`
  (`src/trade_trace/reports/calibration.py:239-247`); at 1..min_sample-1 it
  populates it (`:248-258`). The contract withholds a suggestion when there is
  *no* data but emits a precise one at *one* sample — so a consumer must
  special-case "warned-but-non-null" vs "null". Filed (not fixed) because nulling
  sub-threshold suggestions is a report-output contract design decision.
