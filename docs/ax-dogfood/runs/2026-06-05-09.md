## Friction found & fixed (write-vein audit)

The same auto-derived-schema class (AX-051/054/055/056): a tool registered with
`**_examples_for(...)` and no explicit `json_schema` gets its MCP schema derived
from `example_minimal`, silently dropping runtime-enforced enums/params.

| id | tool | surface | severity | disposition |
|----|------|---------|----------|-------------|
| AX-059 | `journal.fixture_seed` | schema | low | **fixed** `9f3f80e` |
| AX-060 | `memory.retain` | schema | med | **fixed** `9f3f80e` |
| AX-061 | `strategy.update` | schema | med | **fixed** `9f3f80e` |

- **AX-060 `memory.retain`** — `node_type` advertised as a bare `string` though
  the runtime enum-validates it against `NODE_TYPES`
  `{observation,reflection,playbook_rule}` (memory.py:365, self-doc error). The
  highest-value instance yet: a bot calls `memory.retain` every loop and could
  not discover the 3 valid node kinds (nor the
  `importance`/`confidence_base`/validity/`meta_json` knobs) without triggering a
  VALIDATION_ERROR. Fix: `_MEMORY_RETAIN_SCHEMA` mirroring the runtime.
- **AX-061 `strategy.update`** — hid the `status` enum `{active,archived}`
  (runtime self-doc error, the AX-055 sibling) *and* the `hypothesis`/`meta_json`
  update fields, *and* falsely marked `description` **required** on a
  partial-update tool (a bot updating only `status=archived` was wrongly
  blocked). Fix: `_STRATEGY_UPDATE_SCHEMA` — `strategy_id`+`idempotency_key`
  required, all update fields optional, `status` enum advertised.
- **AX-059 `journal.fixture_seed`** — `target` advertised as a bare *required*
  string though the runtime enum-validates it against `FIXTURE_TARGETS`
  (`{mvp-eval-pm, forecast-only-pm, mvp-eval-rich, agent-continuity-loop}`,
  proven live) and *defaults* it to `mvp-eval-pm` (so it is actually optional —
  a second, defaulted-args drift). Fix: `_FIXTURE_SEED_SCHEMA` (enum + optional).

All three are additive; runtime handlers unchanged. **Effective next fresh
session** for the live MCP surface (the constants are in the registry this
session, but the long-lived server holds the old schemas).
