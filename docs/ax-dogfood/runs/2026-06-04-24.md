## Friction found this run

### NEW — AX-044 (appended to scx8): `report.exposure_anomalies` blind to same-side fragmentation

- **Surface:** `report` (`report.exposure_anomalies`) — not previously in the registry.
- **Severity:** low–medium (silent: a bot reads "clean" while fragmented exposure sits open).
- **Observed:** journal holds **two open same-side paper positions** on each of two
  instruments (`report.pnl`: `mkt_7tACn4aqhw6y8BwN` open_count:2 YES,
  `mkt_pwuUfbLMcvhkia50` open_count:2 NO), yet `report.exposure_anomalies`
  returned `count:0` / `codes:[]`.
- **Root cause:** `src/trade_trace/reports/tool_handlers/portfolio_exposure.py:390-400`
  — the `DUPLICATE_DECISIONS` query `GROUP BY (instrument_id, type, side,
  quantity, price, run_id) HAVING COUNT(*) > 1`. It only catches **exact-replay**
  duplicates (identical size + price + run). scx8 fragments differ in
  quantity/price across runs → separate buckets → never flagged. No
  same-side-fragmentation code exists.
- **Cleanest proof:** `mkt_pwuU`'s two NO `paper_enter`s share
  instrument/type/side/**price (0.49)**/run_id (NULL) and differ **only** in
  quantity (100 vs 10) — both open — yet are not flagged purely because
  `quantity` is in the GROUP BY. (`mkt_7tACn4`: 100@0.54 `dec_g094` vs 20@0.37
  `dec_RnrT`.)
- **Disposition:** design-level (add a `FRAGMENTED_SAME_SIDE_EXPOSURE` anomaly
  and/or drop `quantity`/`price` from the `DUPLICATE_DECISIONS` grouping;
  threshold + double-count semantics + overlap with the decision-time
  `already_exposed` advisory) → **appended to `trade-trace-scx8`**, not direct-fixed.
  Lesson retained live as `mem_hg3_nNifTi4u93ly`.
