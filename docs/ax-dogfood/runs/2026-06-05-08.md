## Friction found

| id | surface | severity | one-line |
|----|---------|----------|----------|
| AX-055 | schema (`strategy.upsert`) | med | `status` enum {active,archived} (+ description/hypothesis/meta_json) hidden from the auto-derived schema; runtime enum-validated. |
| AX-056 | schema (`import.commit`) | low-med | `transaction_mode` enum {single,per_row} advertised as a bare string; runtime self-documenting error. |
| AX-057 | schema/tool (`review.bundle`) | med | Registered with no json_schema/example → **zero advertised params**, though the runtime `ReviewBundleInput` accepts a `filter` (+ knobs); passing `filter` even failed `dict_type` (bridge stringified the undeclared param). MCP bot can't scope a reviewer bundle. |
| AX-058 | tool/catalog (`playbook.*`) | med | `report.playbook_adherence` populated path unreachable via MCP: the only version-minting tool (`playbook.propose_version`) is folded/hidden while the catalog-visible `playbook.record_adherence` hard-requires a `playbook_version_id`. Secondary: `playbook.upsert` `status` is unvalidated free-text. |

All four are the same root class as AX-051 (memory.link) / AX-054 (outcome.add):
tools registered with `**examples_for(...)` and no explicit `json_schema` get
their MCP schema auto-derived from `example_minimal`, silently dropping enum
constraints and extra params.
