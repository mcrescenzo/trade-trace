### AX-045 — declared-risk chain verified live end-to-end

Last run's fix (903f9f4) was unit-verified only. This fresh session's MCP server
boots the fixed code, so I drove the whole chain on live data:

1. **Schema (discovery side).** `tool.schema decision.add` now advertises
   `declared_risk_amount` / `declared_risk_unit` in `json_schema.properties`
   (with a description pointing at report.risk/opportunity and the
   `missing_risk_budget` consequence) **and** in the
   `paper_enter`/`actual_enter`/`add` `x-decision-matrix` optional lists, plus an
   `actual_enter` example carrying `declared_risk_amount:38`. A schema-following
   bot can now discover the field.
2. **Write → position.** `decision.add(paper_enter, declared_risk_amount=0.94,
   declared_risk_unit=dollar)` opened `pos_QE8HbdO-_dDXV875`. `report.open_positions`
   shows the row with `initial_risk_amount:0.94` and **no `missing_risk_budget`
   caveat** (only the `no_strategy` info caveat) — the declared budget flowed
   through and cleared the per-row flag.
3. **report.risk (the previously-dead surface).** Now returns
   `n_decisions_with_risk:1`, `n_pending_with_risk:1`, `pending_risk_count:1`,
   and the caveat *"1 decision(s) carry declared risk but have no closed/resolved
   position P&L yet."* The R-multiple aggregate stays `null`/`sample_size:0` —
   **correct**, because R-multiples need *closed* positions and paper positions
   never close (intentional position-lifecycle #1). This also resolves the
   residual the 903f9f4 run flagged ("report.risk still silent on an all-open
   journal"): it is no longer silent — it surfaces pending declared risk; only
   the *realized* R aggregate awaits closes, which is by design.

**Verdict:** AX-045 is genuinely fixed for an MCP-only bot. Promote to
`fixed (verified live)` in the registry.

Minor observation (not filed): `report.risk` accepts a `filter` argument in its
MCP schema but `supported_filter_paths:[]` — it rejects *every* filter field
(e.g. `instrument.instrument_id`) with a self-documenting `VALIDATION_ERROR`
that lists the supported set as empty. It is a global R-multiple aggregate by
design; the error is honest. Borderline-intentional; left as an observation to
avoid backlog noise, consistent with prior report.risk residual handling.
